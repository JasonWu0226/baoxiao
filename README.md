# 自动化报销软件 使用说明

## 功能介绍

本软件可以帮您自动化处理报销流程：

1. **解析淘宝订单** - 从"已买到的宝贝.pdf"提取订单信息
2. **解析微信支付记录** - 从微信支付交易明细PDF提取消费记录
3. **解析美团订单** - 从美团截图提取订单信息（需要OCR支持）
4. **邮箱发票提取** - 从QQ邮箱自动提取指定时间范围内的发票（需要配置）
5. **生成报销Excel** - 基于模板自动生成符合格式的报销表
6.   下载发票的功能需要将所有发票的邮件进行发票下载，下载完成之后发票时间是指定时间之后才保留，指定时间之前的发票全部过滤，这个可以参考之前的邮件下载，但是人工确认的动作需要在软件中确认清楚，并保留记录方便后续升级自动操作
7、需要在软件中进行是否需要报销动作确认、有些存在发票的但是具体分不清楚匹配哪张发票的需要在软件内可以选择的方式进行闭环，
8、这里存在订单金额和发票金额有差异的情况、一个订单多张发票、多个订单一个发票的情况
9、归档已经确认好了的项单独放置在一个文件夹内，里面包含了订单信息和截图、以及发票，Excel里面背景切换成绿色，待确认是黄色
10. **上期报销索引** - 先基于上期发票和报销文档生成查询表，邮箱下载前自动标记疑似上期已报销邮件，避免重复下载和重复确认

## 开发原则

这个软件的目标不是让 Codex 代替人工临时处理一次报销，而是把当前遇到的问题逐步固化成软件能力。后续开发遵循以下原则：

1. 所有报销处理动作必须能在软件里完成，包括发票下载、解压、去重、查漏、上期重叠核验、订单匹配、截图归集、报销 Excel 生成。
2. 所有人工确认必须在软件里完成，并保存到规则记录中，不能依赖在 Excel 里临时标注或在 Codex 里临时判断。
3. 所有处理结果必须可查询、可重跑、可增量处理，已经确认正确的项目后续不重复打扰。
4. Codex 的职责是开发和升级软件，不作为日常报销流程的一部分。
5. 如果某个问题当前只能靠 Codex 处理，说明软件还缺少入口，需要优先开发成按钮、页面、批量操作或核验报表。

## 推荐流程

1. 把上期材料放到 `报销准备的资料/上一期报销`，或在软件顶部填写 `上期发票目录`。
2. 打开软件后先点击 `更新上期报销索引`。
3. 软件生成 `输出/上期报销查询表_时间.xlsx` 和 `规则/上期报销索引.json`。
4. 再执行邮箱发票下载，命中上期索引的邮件会标记为 `疑似上期已报销跳过`。
5. 软件会按发票 PDF/XML 内的开票日期过滤，早于开始日期的文件会标记为 `发票日期不符` 并归档。
6. 点击 `归集PDF发票并汇总` 后，查看 `识别问题诊断` 页确认哪些文件是真正识别失败、哪些只是行程单或上期重叠。
7. 只对剩余新发票、异常链接、未匹配项目做人工确认。

## 配置说明

首次使用可以复制 `matcher_config.example.json` 为 `matcher_config.json`，再填写本机路径、邮箱账号和授权码。`matcher_config.json`、发票、规则记录、输出文件都属于本地隐私数据，默认不会提交到 Git。


## 模板格式说明

模板文件 `报销模版.xlsx` 包含以下表单：

1. **报销明细** - 主要数据表
   - 列2: 开票日期
   - 列3: 发票金额（RMB）
   - 列4: 有无正式发票
   - 列5: 支出事由
   - 列6: 项目名称
   - 列7: 归属研发支出
   - 列8: 经办人
   - 列9: 备注发票号

2. **付款截图** - 没有发票的都需要有付款截图凭证，并且在备注中需要注明需要存在付款截图，然后单独把订单的或者转账的截图放置付款截图页面里

3. **汇总** - 报销汇总统计

## 注意事项

1. **淘宝订单**：只处理"交易成功"的订单，跳过"交易关闭"的订单
2. **微信支付**：只处理"商户消费"和"扫二维码付"，跳过收入
3. **美团订单**：当前版本需要手动处理或提供OCR支持
4. **邮箱发票**：需要从我的QQ邮箱中自动下载发票，过滤掉重复的发票，有些里面是下载链接还要点击进去才能下载
